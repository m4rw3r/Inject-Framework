The Boot Process

A walkthrough through the boot process of the Inject Framework explains a lot about how the Inject Framework is functioning and how the different components are initialized.

Step 1: The Front Controller

The Front Controller is a PHP file which initially will receive the request. It will load the Inject class file, call Inject::init(), configure it, create a request object and eventually call Inject::run().

// load Inject Framework require './Inject/inject.php'; // initialize the autolader and error handlers Inject::init(); // configure Inject::set_config_file('./config.php'); Inject::set_config('inject.front_controller', basename(__FILE__)); // create request object and run Inject::run(new Inject_Request_HTTP);

(Step 2: The Inject::init() method)

The Inject::init() method is not modifiable by the user in any way, and it does the following:

  • Registers an autoloader
  • Registers error and exception handlers
  • Starts output buffering

(Step 3: The Inject::run() method)

The Inject::run() method is not modifiable by a user component and will handle the output buffering, the creation of the dispatcher and finally it will destroy the request, after the controllers have finished.

Step 4: The dispatcher

The Dispatcher is a user replaceable class whose task is the following:

  1. Get the controller class name from the request object, the dispatcher raises 404 if needed
  2. Get the action from the request object, the dispatcher sets the default
  3. Instantiate the controller object
  4. Call the action

Step 5: The Controller constructor

The controller constructor receives the request object which will contain parameters and information about the request, it is usually assigned to the property $request.
After that, the user can do almost whatever he likes.

Step 6: The Controller Action

The action has even less constraints.

Step 7: The output handler

Lastly, the Inject::parse_output() is called. It has hooks for user functions/methods to apply parsing and / or compression before the output is sent to the browser.
